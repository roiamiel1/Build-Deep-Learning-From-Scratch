# Stage 19: Mini-batch training

**Context** — In `stage_15` you trained with *full-batch* gradient descent: every epoch the whole dataset went through one forward/backward/step. Real training almost never does that — datasets are too big, and full-batch updates are slow and over-smooth. This stage splits the data into **mini-batches**, runs the same forward/`backward()`/`step()`/`zero_grad()` cycle once per batch, and lets you watch the three-way trade-off between **batch size**, **gradient noise**, and **throughput**. You reuse `MLP`, `mse_loss`, `SGD`, `make_moons`/`make_spiral`, and `accuracy` from `stage_15` unchanged.

**Background** — The training loss is an *average over the dataset*, $L(\theta)=\frac{1}{N}\sum_{i=1}^{N}\ell_i(\theta)$. Its true gradient is $\nabla_\theta L=\frac{1}{N}\sum_i\nabla_\theta\ell_i$. A mini-batch of size $B$ drawn (without replacement) from the $N$ examples gives the **stochastic estimate**
$$\hat g_B=\frac{1}{B}\sum_{i\in\text{batch}}\nabla_\theta\ell_i,\qquad \mathbb{E}[\hat g_B]=\nabla_\theta L,$$
so each batch gradient is *unbiased* — on average it points the right way. What changes is the **variance**: for i.i.d. per-example gradients with variance $\sigma^2$, the per-coordinate variance of the estimate is $\operatorname{Var}[\hat g_B]=\sigma^2/B$. Small $B$ ⇒ noisier steps (more exploration, can escape shallow minima, but jittery loss); large $B$ ⇒ smoother, more accurate steps that approach full-batch. Because you **average** inside the batch (the $\tfrac1B$, which is exactly `mse_loss`'s `.mean()` from `stage_15`), the gradient *magnitude* stays comparable across batch sizes, so a fixed learning rate transfers reasonably. One **epoch** = one full sweep over all $N$ examples = $\lceil N/B\rceil$ batches; shuffling each epoch decorrelates consecutive batches. Throughput grows with $B$ (vectorized matmul amortizes Python/loop overhead) up to memory limits — fewer, larger steps per epoch. This is the same `Tensor.backward()` engine (`stage_09`) and `SGD` (`stage_15`); only the *data feeding* changes.

- `iterate_minibatches(X, y, batch_size, *, shuffle=True, seed=None, drop_last=False)`: a **generator** yielding `(X_b, y_b)` NumPy slices. With `shuffle`, permute indices once per call via `np.random.default_rng(seed)`. Cover all `N` rows in order of the (possibly shuffled) permutation; the last batch has size `N % batch_size` unless `drop_last` drops it. `batch_size <= 0` or `> N` and other bad inputs raise `ValueError`. Every yielded pair has matching first dimension.
- `train_minibatch(model, X, y, *, lr, epochs, batch_size, shuffle=True, seed=None, optimizer=None, drop_last=False) -> dict`: for each epoch, iterate mini-batches and run, **in order**, `pred = model(X_b)` → `loss = mse_loss(pred, y_b)` → `loss.backward()` → `optimizer.step()` → `optimizer.zero_grad()`. If `optimizer is None`, build `SGD(model.parameters(), lr)`. Reshape `y_b` to broadcast against `pred` (e.g. `(B,1)`). Return a dict with keys `"batch_loss"` (list of every batch's `float(loss.data)`, in order), `"epoch_loss"` (list, one **size-weighted** mean per epoch over its batches), and `"steps"` (total `optimizer.step()` calls). Per-batch loss is already the *mean* over the batch (`mse_loss`), so the gradient magnitude is comparable across batch sizes.
- `gradient_noise(model, X, y, batch_size, *, n_batches, seed=None) -> float`: with grads at a fixed `model`, estimate gradient noise. Draw `n_batches` random mini-batches; for each, zero grads, forward, `mse_loss`, `backward()`; flatten and stack all `p.grad` into one vector per batch. Return the **mean across parameter coordinates of the per-coordinate variance** of these batch-gradient vectors (a single float). Do not call `optimizer.step()` (the model must not move). Larger `batch_size` must give smaller noise.
- `epochs_to_threshold(history, threshold) -> int`: given an `"epoch_loss"` list, return the **1-based** index of the first epoch whose loss `<= threshold`, or `-1` if never reached.
- `plot_batch_comparison(histories, path=None)`: given `{label: epoch_loss_list}`, Matplotlib-plot each curve (loss vs epoch) on shared axes with a legend; save to `path` if given, else return the figure. Never plot inside the training functions.
